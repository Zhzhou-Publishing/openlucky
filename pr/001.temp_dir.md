# 需求：工作目录 temp 化

## 使用 tmp 库

- 使用 tmp 库而不是裸文件操作控制 temp 目录的生命周期。`npm install tmp`。

## 改造 IPC 事件

- 取消 'get-images' 这个 IPC 事件里面提前转码 raw 格式的所有代码。它只做一件事，就是把符合 imageExtensions 和 rawExtensions 都列举出来。保留 'get-images' 里面从 .preset.json 获取已处理文件的能力。

- 取消 'get-full-res-image' 这个 IPC 事件从 temp 目录读取的代码。它只做一件事，就是从输入参数的 directoryPath 里面获取到 filename 并返回。

- 'apply-preset' 这个 IPC 事件的参数调整为 inputPath，outputPath, preset，执行 openlucky filmbatch --input <inputPath> --output <outputPath> --preset <preset>

- 'apply-filmparam' 这个 IPC 事件的参数调整为 inputPath, outputPath, filename, params，相应调整 openlucky filmparam 这个命令的参数。

- 'apply-filmparambatch' 这个 IPC 事件的参数调整为 inputPath, outputPath, params，相应调整 openlucky filmparambatch 这个命令的参数。

## 改造页面

### PhotoGallery.vue 改造

- 在 PhotoGallery.vue 的 onMounted 事件里面就创建好一个 temp 目录作为 workingDirectory，所有的文件操作都将放入临时目录。

- 处理非 RAW 格式的图片到 workingDirectory：PhotoGallery.vue 的 loadImages 函数里面，将非 raw 格式的文件直接复制到 workingDirectory 里面。

- 处理 RAW 格式的图片到 workingDirectory：PhotoGallery.vue 的 loadImages 函数里面，将 RAW 格式的图片文件用 openlucky raw2tiff -i raw_pic_file_111.raf -o <workingDirectory>/raw_pic_file_111.raf.tif 转码放入 workingDirectory 里面。

- 处理其他文件到 workingDirectory：复制 .preset.json 文件到 workingDirectory。

- 禁止复制除了上面提到的 RAW 格式图片、非 RAW 格式图片、.preset.json 文件以外的其他文件，特别禁止复制任何子目录。

- loadImages 事件最后得到的结果是上面 temp 目录的 workingDirectory，整个页面将拉取这个目录里的所有文件。在 loadImages 执行完成前，整个页面保持 loading 状态。

- 创建一个 outputDirectory，设置值为 path.join(directoryPath, 'output')（path 通过 window.require('path') 导入）。

- applyPreset 操作 apply-preset 传入参数：inputPath: workingDirectory，outputPath: outputDirectory, preset: selectedPreset

- 当 push 到 PhotoEdit 的时候，加一个参数 outputDirectory。

### PhotoEdit.vue 改造

- 创建 outputDirectory，值来自同名 param 参数。

- apply 操作调用 'apply-filmparam' 的时候，输入参数为：inputPath: workingDirectory, outputPath: outputDirectory, filename: imageName, params: params。

- applyAll 操作调用 'apply-filmparambatch' 的时候，输入参数为：inputPath: workingDirectory, outputPath: outputDirectory, filename: imageName, params: params。
