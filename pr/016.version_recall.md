# 版本召回

## 当前处理版本升级的方法

以下信息供参考，因为是已经是完成的需求，所以不作为本次需求的编码点：

当前使用下面的 Github API 来判断当前版本是不是和 latest 版本一致，如果不一致则弹窗提示用户升级软件。

https://api.github.com/repos/Zhzhou-Publishing/openlucky/releases/latest

## 即将编码的功能要达到的效果

除了上面对照 latest 版本和当前版本以外，还查找当前版本是否被开源仓库删除，如果已经删除则表示版本被作者召回了，应当禁止用户再使用当前版本。

被拦截使用的用户，软件应当强制退出无法使用。

## 如何检查当前版本是否被召回？使用 Github API

当前版本号用于用下面的 API 查询指定版本：

比如当前版本号是：1.4.2-rc9，那么 tag=v1.4.2-rc9（前缀不带v则加一个v）

https://api.github.com/repos/Zhzhou-Publishing/openlucky/releases/tags/{tag}

如果发现 HTTP 响应是 404，则拦截当前版本，强制让用户下载最新版本：

404 参考 response body 如下：

```json
{
  "message": "Not Found",
  "documentation_url": "https://docs.github.com/rest/releases/releases#get-a-release-by-tag-name",
  "status": "404"
}
```

将上述的对照过程（包括访问了哪个链接、得到什么结果）详细打印到日志。

## 如何拦截当前版本

请注意：只有在上面的判断已经 404 过后才执行这个强制退出，否则还是按以前的方式允许用户继续使用当前版本。

现有的弹出更新提示链接，弹窗如果点击取消，则退出整个主进程。

文案上加上一句话：当前软件版本已经不再有技术支持，请下载最新版本。

## 弹窗多语言支持

由于目前不方便在页面外做 i18n，所以弹窗的文案准备两个语言版本。

现在的语言是英文，请多准备一套简体中文的文案。

如 electron 引擎发现当前语言是简体中文（zh_CN，zh_Hans）则使用简体中文的文案。
