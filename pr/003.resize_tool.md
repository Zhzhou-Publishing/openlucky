# 缩放图片子命令

## 新增子命令 openlucky > tool > resize

1. 为 cmd/openlucky.py 新增 tool 子命令，再在 tool 下再新增 resize 子命令。
2. resize 子命令负责重定义图片大小，参数如下：
   1. --input,-i 指定文件地址，支持 jpeg png bmp 等位图，也支持 tiff 图片和 RAF 等 RAW 格式图片。
   2. --output,-o 指定输出文件地址，非 RAW 格式的图片需与原图保持同文件名。如果是 RAW 格式图片，可以是原文件名，也可以是原文件名为前缀的 tiff 格式图片，比如输入 DSA00341.RAF，合法的输出文件名是：DSA00341.RAF，DSA00341.RAF.tif，DSA00341.RAF.tiff，不合法的文件名如：DSA00341.RAF.jpg，DSA00341.png 等。最终非 RAW 格式的图片输出与原格式相同的图片，RAW 格式输出一个 tiff 格式的图片。
   3. --edge,-e 指定哪条边作为缩放依据。枚举值 {width,height,long-edge,short-edge}，表示按宽、高、长边、短边为缩放依据。默认值是 long-edge。
   4. --mode,-m 指定缩放模式，枚举值 {ratio, fixed-value}，表示按比例缩放，按固定值缩放。默认值是 fixed-value。
   5. --value,-v 指定缩放值。如果缩放模式是 ratio，则支持输入一个 (0,1] 区间的浮点数。如果缩放模式是 fixed-value，则支持输入一个 (0,+Inf) 区间的一个整数表示缩放到多少具体的像素值。
3. 具体样例如下：
   1. openlucky tool resize -i 1.png -o 1.jpg -v 100。报错，因为前后输入输出文件名不合规，不满足【非RAW格式文件名一致、RAW格式为原文件名或原文件名+tiff/tif后缀】的规则。
   2. openlucky tool resize -i 1.png -o 1.png -v 100。将原文件长边缩放到 100px，短边按比例缩放。
   3. openlucky tool resize -i 1.png -o 1.png -e height -v 100。将原文件高缩放到 100px，宽按比例缩放。
   4. openlucky tool resize -i 1.png -o 1.png -e height -m ratio -v 0.5。将原文件高缩放到原来大小的 50%，宽按比例缩放。
   5. openlucky tool resize -i 1.png -o 1.png -e height -m ratio -v 200。报错，ratio 模式下只允许输入 (0,1] 区间的值。
   6. openlucky tool resize -i 1.png -o 1.png -e height -m fixed-value -v 0.5。报错，fixed-value 模式下只允许输入 (0,+Inf) 区间的整数值。
4. 核心函数放在 lib/tool 包下新建的文件 resize.py 里。
