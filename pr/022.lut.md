# 快速查找表 LUT 实现快速运算

## LUT 应用函数

CLI 实现一个套用 LUT 的中间函数 `apply_lut`，签名：

```python
apply_lut(function_name, image, **kwargs)
```

参数：

- `function_name`：需要调用的函数名字（必须已注册到 `FUNC_REGISTRY`）。
- `image`：作用对象，float32 numpy 数组，值域 [0, 1]。
- `**kwargs`：传给原始函数的具名参数；同时用来拼出 LUT 文件名中的 `param_str`。

执行过程：

1. **拼 `param_str`**：把 kwargs 按顺序组合为 `key-dotted_value` 段，`-` 连接。其中 `dotted_value` 用 `f'{v:.2f}'.replace('.', '_')` 强制保留两位小数（与 `openlucky.dev genlut` 生成的文件名格式保持一致）。
   - 样例：`p=1.1, k=1.5, s=0` → `p-1_10-k-1_50-s-0_00`。
2. **定位并加载 LUT**：在 `lut/` 目录下查找 `com.github.zhzhou-publishing.openlucky.<function_name>.<param_str>.lut`。
   - `lut/` 目录定位：开发态为仓库根目录下的 `lut/`；发布安装后为 CLI 可执行文件（`bin/openlucky`）的兄弟目录 `lut/`。通过 `getattr(sys, 'frozen', False)` 判断当前运行模式。
   - LUT 是 65536 项 `uint16` 文件（由 genlut 输出），用 numpy 索引应用：`lut[clip(image*65535).astype(uint16)] / 65535.0`，结果为 float32 [0, 1]。OpenCV 的 `cv2.LUT` 仅支持 8-bit/256 项 LUT，对 16-bit 全精度场景不适用，所以这里改走 numpy。
3. **回退**：找不到 LUT 文件、或 LUT 大小不是 65536 时，回退调用 `FUNC_REGISTRY[function_name](image, **kwargs)`。

## FUNC_REGISTRY 抽离

将 `FUNC_REGISTRY` 从 `cli/openlucky.dev.py` 抽到 `cli/lib/lut.py`，与 `apply_lut` 共处一个模块，方便其它脚本（包括现有的 `openlucky.dev.py`）直接 `from cli.lib.lut import FUNC_REGISTRY` 使用。

## 范围

本次只实现 wrapper 函数与 `FUNC_REGISTRY` 抽离，不涉及现有 CLI 命令（`film` / `filmparam` / `curve levels` 等）的接入；接入工作另行规划。
