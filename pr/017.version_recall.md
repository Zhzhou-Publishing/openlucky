# 版本召回-增强功能

## 增强功能

为避免 Github API 资源被浪费，合理利用以往写入的 lastUpdateCheck.txt 文件的功能，我在此做出以下增强设计：

1. 读取 lastUpdateCheck.txt 的内容，如果里面写入的内容是 recall:$version，并且 $version 是当前的版本，则直接弹出版本召回的弹窗。
2. 如果 $version 部分不是当前的版本，或者是由一个 latest 版本对照判断罗辑写入的时间戳，则继续进行 latest 版本对照，做常规的时间的判断，未来将被 latest 版本的时间判断覆写掉。
3. recall:$version 这个字符串由召回判断的逻辑写入，如果当前版本需要召回，那就将【recall:$version】写入到 lastUpdateCheck.txt 文件中。
4. recall:$version 的 $version 部分是加了 v 前缀的版本号，比如某次需要召回的当前版本是 1.4.2-rc11，那么写入进文件 lastUpdateCheck.txt 的就是 recall:v1.4.2-rc11，这样做是为了避免误判而将未来所有新版本都杀掉。
