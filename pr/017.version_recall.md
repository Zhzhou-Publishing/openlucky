# 版本召回-增强功能

## 现有功能简述

对当前版本的升级判断分三部分：【升级时机检查】、【召回判断】和【latest 版本对照】。

当前仅有【latest 版本对照】的时候会写入  lastUpdateCheck.txt。

为避免 Github API 资源被浪费，合理利用以往写入的 lastUpdateCheck.txt 文件的功能，我在此做出以下增强设计：

## 增强功能

1. 召回判断的时候就读取 lastUpdateCheck.txt 的内容，如果里面写入的内容是 recall:$version，并且 $version 是当前的版本加 v 前缀的字符串（如果版本号不带 v 前缀才加 v 前缀），则直接弹出版本召回的弹窗，而不进行后续的召回判断和升级判断。
2. 如果 $version 部分不是当前的版本，或者是由一个 latest 版本对照判断罗辑写入的时间戳，则继续进行召回判断、latest 版本对照。
3. recall:$version 这个字符串由召回判断的逻辑写入，如果当前版本需要召回，那就将【recall:$version】写入到 lastUpdateCheck.txt 文件中。
4. recall:$version 的 $version 部分是加了 v 前缀的版本号，比如某次需要召回的当前版本是 1.4.2-rc11，那么写入进文件 lastUpdateCheck.txt 的就是 recall:v1.4.2-rc11，这样做是为了避免误判而将未来所有新版本都杀掉。

## 参考流程

1. 升级时机检查：读取 lastUpdateCheck.txt，首先判断是否需要召回，如果需要召回则直接弹出召回的弹窗，如果不是要召回的版本，则对照时间是否升级。
2. 召回判断：调用 Github API，判断当前版本是否存在。如果当前版本不存在则弹召回弹窗，并写入 recall:$version 到 lastUpdateCheck.txt 文件并用 recalled 这个全局变量告知后续流程已经被召回。
3. latest 版本对照：调取 Github API，对照 latest 版本是否与当前版本相同，如果不同则弹出需要升级的弹窗，最后判断 recalled 全局变量，如果是 false 则写入当前时间戳到 lastUpdateCheck.txt。
