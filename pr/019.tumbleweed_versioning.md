# 风滚草版本控制

## 设计版本规则

| 版本后缀类型 | 含义   | 可以滚动到哪些版本   |
| ------------ | ------ | -------------------- |
| 无后缀       | 正式版 | 下一个无后缀的正式版 |
| rc           | 预览版 | 下一个预览版         |
| beta         | 公测版 | 下一个公测版         |
| alpha        | 内测版 | 下一个内测版         |

## 版本滚动升级

1. 先判断当前版本是哪种类型。
2. 通过现有检查对照版本的 latest Github API 查询 latest 版本的版本号后缀，和当前版本的版本号后缀作对比。
3. 如果 latest 版本的版本号后缀和当前版本的版本号后缀相比符合上表对应到【可以滚动到哪些版本】才向用户弹窗推送这个 latest 版本。
